# Docker Networking with Isolated Containers - Advanced Project

================================================================================
1. INNOVATIVE FEATURES & SECURITY HARDENING
================================================================================

## Core Security Features:
- Network Isolation: Custom bridge networks for frontend-to-backend and backend-to-db communication.
- Internal DB Network: The `backend_net` is set to `internal: true`, preventing the database from reaching the external internet.
- Read-Only Root Filesystem: Frontend service has a read-only root FS, reducing attack surface.
- Resource Limits: Strict CPU (e.g., 0.10) and Memory (e.g., 64MB) limits to prevent resource exhaustion/DoS.
- Health Checks: Automated health monitoring for all services (frontend, backend, and db).
- High Availability: `restart: always` policy for the backend service.

## Automation & Testing:
- Makefile Integration: Professional-grade CLI for managing the entire lifecycle.
- Python Test Suite: Robust network connectivity validation with JSON reporting.

================================================================================
2. PROJECT ARCHITECTURE
================================================================================

- [Frontend] (on frontend_net) -> [Backend] (on frontend_net & backend_net) -> [DB] (on backend_net)
- DB is COMPLETELY isolated from the Frontend.
- Only the Backend can bridge the communication between both networks.
