1. PROJECT OVERVIEW
-------------------
This is a full-stack "Track Placement Preparation" platform containerized using DevOps best practices.
It features a 3-tier microservices architecture:
- **Frontend Tier**: React (Vite) served by Nginx (Public Network).
- **Backend Tier**: Node.js/Express API (Internal Logic).
- **Database Tier**: MongoDB (Isolated Secure Network).

2. ADVANCED DEVOPS FEATURES
---------------------------
- **Multi-Tier Isolation**: The Database (MongoDB) is in its own private network (`db-network`).
- **Nginx Reverse Proxy**: All API calls are routed through Nginx to prevent CORS issues and hide the backend.
- **Data Persistence**: Uses Docker Volumes to ensure that all placement data is saved even if containers are deleted.
- **Docker Orchestration**: The entire platform is deployed with a single `docker-compose up` command.
- **Health Checks**: Automated monitoring for container availability.

4. VIVA QUESTIONS (ADVANCED)
----------------------------
Q: Why use Nginx in a containerized React app?
A: Nginx provides a robust, production-grade server for static files and handles reverse proxying to avoid cross-origin (CORS) errors.

Q: What is the benefit of a multi-tier network?
A: It provides "Defense in Depth". Even if the frontend is compromised, the attacker cannot reach the database directly.

Q: How does Docker Compose help in development?
A: It provides a "reproducible environment" where everyone on the team has the exact same setup, avoiding "it works on my machine" issues.
